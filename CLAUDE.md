# hanashitara プロジェクト - 開発ガイドライン (Claude Code 用)

このファイルは、開発を担当する Claude Code が常に読み込み、システムの仕様やユーザー（はんださん）の意図を正確に反映するためのマスターファイルです。指示を実行する前に、必ず以下のルールとコンテキストを前提としてください。

## 1. 開発とデプロイの絶対ルール

- **Vercelプロジェクトの複数構成の認識:**
  このリポジトリは Vercel 上で `hanashitara` (No Production Deployment) と `hanashitara-system` (`hanashitara-system.vercel.app`) の2つのプロジェクトに紐付いています。
  コード変更をプッシュした際は、「プッシュしました」で終わらせず、**必ず `hanashitara-system` 側で再ビルドが成功し、本番環境 (`https://hanashitara-system.vercel.app/`) に変更が反映されたことまでを確認してからタスクを完了**してください。

- **GAS (Google Apps Script) の更新手順:**
  GAS側のコードを変更した際、絶対に「新しいデプロイ」を選んでURLを変更してはいけません。フロントエンドとの通信が壊れます。
  **【正しいGAS上書き手順】: 「デプロイを管理」を選択 → 左側の現在アクティブなデプロイを選択して「鉛筆マーク（編集）」を押す → バージョンを「新バージョン」に変更して「デプロイ」を押す。** これを厳守してください。

- **GAS コード変更時は必ずクリッカブルリンクを提示する（毎回・例外なし）:**
  `GAS_LATEST_FOR_PASTING.txt` を編集した場合（= GAS側の更新がある場合）、**完了報告の中で毎回必ず** 以下の書式でリンクを貼る。この書式ならはんださんがクリック一発で開けてコピーできる（検証済み）。
  - **【必須書式】**: `[GAS_LATEST_FOR_PASTING.txt](GAS_LATEST_FOR_PASTING.txt)`
    → Claude Code の UI で下線付き青リンクとして表示され、クリックでファイルが開く
  - NG書式: `[...](file:///C:/Users/.../GAS_LATEST_FOR_PASTING.txt)` や絶対パスをコードブロックで貼るだけ（どちらもクリック不可）
  - タイミング: commit・push・Vercel 確認が終わった後、完了報告の最後に「**📂 GAS コード全文:** [GAS_LATEST_FOR_PASTING.txt](GAS_LATEST_FOR_PASTING.txt)」の形で貼る
  - フロントのみの変更で GAS に触っていない場合は不要

- **バグ修正の原則:**
  その場しのぎのハックなコード修正（一時的なパッチ等）は行わず、必ず根本原因を特定して解決してください。ファイルの行数が多く肥大化している場合は、コンテキストを見失わないよう注意深く検索・置換を行ってください。

## 2. UI とデザインのルール

- **白ベース・AppleライクなUIの徹底:**
  UIは「白系ベース、Apple UI を参考にしたシンプルで洗練されたデザイン」を統一ルールとします。一部の要素を目立たせるために、安易に全体のトンマナを崩すダークカラー（`from-gray-900`等）を使わないでください。ボーダー、微細なグレー背景（`bg-gray-50`）、シャドウなどを用いて美しく整理してください。

- **スマートなUIコンポーネント:**
  余計な入力欄やボタンがUIを圧迫しないよう、適切な「開閉式モーダル（ダイアログ）」や「スクロール選択」「シンプルな表記」を活用し、一画面に表示される情報量を整理・洗練させてください。

- **変更時の即時フィードバック (UX):**
  設定変更などのアクションを行った際は、ラグを感じさせないようにUIの即時更新（オプティミスティック更新）を行うか、トースト通知（「保存しました」等）を表示し、ユーザーが不安にならないようにしてください。

## 3. 実装の進め方

- TypeScriptの型エラー（Lintエラー）を残さないこと。ステート（`useState`等）を追加・削除した際は、依存する他のロジックが壊れていないか全体をチェックすること。
- 何か変更を行った際は、必ず自分のコード変更結果を確認し、「システムとして機能するか」「デザイン要件を満たしているか」をテストしてからユーザーへ完了報告を行ってください。
